Model Setup for Linux/Docker

1. Gemini

Gemini model is setted up by default, just enter your API key in setting > API Management.

2. ChatGPT (OPENAI)

  1. Enter kuwa-aios/docker/compose directory and copy the .yaml file.
cp gemini.yaml chatgpt.yaml

  1. Edit chatgpt.yaml:

  2. line 2: change gemini-executor to chatgpt-executor

  3. line 8: change geminipro to chatgpt
  4. line 9: change geminipro to gpt-4-turbo or other non-repeated code
  5. line 10: change Gemini to OpenAI GPT-4 or other name that will be shown as Kuwa web server chatroom name
  6. line 11: change gemini.png to chatgpt.png

  7. line 15: change command: ["--api_key", …… to command: ["--model", "gpt-4-turbo","--api_key", "<FILL IN YOUR API key>"

  8. Edit run.sh:

Add chatgpt into confs array and execute run.sh again.

confs array:

chat room interface:

3. TAIDE

  1. Download the TAIDE 8B 4bit version gguf file taide-8b-a.3-q4_k_m.gguf based on Llama3 from here.

```sh= cd ~ mkdir gguf/taide curl -L -o "taide/taide-8b-a.3-q4_k_m.gguf" https://huggingface.co/nctu6/Llama3-TAIDE-LX-8B-Chat-Alpha1-GGUF/resolve/main/Llama3-TAIDE-LX-8B-Chat-Alpha1-Q4_K_M.gguf?download=true


2. Enter `kuwa-aios/docker/compose` directory and copy the .yaml file.
```sh
cp llamacpp.yaml llamacpp-taide.yaml

  1. Edit llamacpp-taide.yaml:

  2. line 2: change llamacpp-executor to llamacpp-taide-executor

  3. line 9: change TAIDE 4bit to llamacpp-taide or other non-repeated code
  4. line 10: change Gemini to Llama3-TAIDE-LX-8B-Chat-Alpha1-4bit or other name that will be shown as Kuwa web server chatroom name
  5. line 15: change command: ["--model_path", "/var/model/taide-4bit.gguf" ...... to command: ["--model_path", "/var/model/taide-8b-a.3-q4_k_m.gguf" ......

  6. line 17: change /path/to/taide/model.gguf to your path to the gguf file, and change /var/model/taide-4bit.gguf to /var/model/taide-8b-a.3-q4_k_m.gguf

  7. Edit run.sh:

Add llamacpp-taide into confs array and execute run.sh again.

If you faced the error ModuleNotFoundError: No module named 'llama_cpp', please refer to the commit to fix it.

4. Others, using Ollama

  1. Use Ollama, a simple API for running and managing models, to pull model.
ollama pull <model name>
  1. Enter genai-os/docker/compose directory and copy the .yaml file.
cp gemini.yaml ollama-<name>.yaml

  1. Edit ollama-<name>.yaml:

  2. line 2: change gemini-executor to ollama-<name>-executor

  3. line 8: change geminipro to chatgpt
  4. line 9: change geminipro to <access code>, recommended a non-repeated model version name
  5. line 10: change Gemini to <the code that will be shown in chat room interface>
  6. line 15: change command: ["--api_key", …… to ["--model", "<model name>", "--base_url", "http://host.docker.internal:11434/v1", "--api_key", "ollama"]

Note that the API URL originally prompted in Ollama is http://<localhost>:11434/v1, but since we are using Docker to load this LLM, we can't directly access services through localhost in the container. Therefore, we should change localhost to the machine IP address or use Docker-provided IP host.docker.internal to access to services.

  1. Edit run.sh:

Add <yaml file name> into confs array and execute run.sh again.

  2. Enter genai-os/docker/compose directory and copy the .yaml file.

cp gemini.yaml lmstudio-<name>.yaml

  1. Edit lmstudio-<name>.yaml:

  2. line 2: change gemini-executor to lmstudio-<name>-executor

  3. line 8: change geminipro to chatgpt
  4. line 9: change geminipro to <access code>, recommended a non-repeated model version name
  5. line 10: change Gemini to <the code that will be shown in chat room interface>
  6. line 15: change command: ["--api_key", …… to ["--model", "<model name>", "--base_url", "http://host.docker.internal:11434/v1", "--api_key", "lm-studio"]

Note that the API URL originally prompted in Ollama is http://<localhost>:11434/v1, but since we are using Docker to load this LLM, we can't directly access services through localhost in the container. Therefore, we should change localhost to the machine IP address or use Docker-provided IP host.docker.internal to access to services.

  1. Edit run.sh:

Add <yaml file name> into confs array and execute run.sh again.

6. Others, using GGUF

  1. Download the gguf file.

  2. Enter genai-os/docker/compose directory and copy the .yaml file.

cp llamacpp.yaml llamacpp-<name>.yaml

  1. Edit llamacpp-<name>.yaml:

  2. line 2: change llamacpp-executor to llamacpp-<name>-executor

  3. line 8: change taide-4bit to llamacpp-<name>
  4. line 9: change TAIDE 4bit to <access code>, recommended a non-repeated model version name
  5. line 10: change Gemini to <the code that will be shown in chat room interface>
  6. line 15: change command: ["--model_path", "/var/model/taide-4bit.gguf" ...... to command: ["--model_path", "/var/model/<gguf file name>" ......

  7. line 17: change ["/path/to/taide/model.gguf to your path to the gguf file, and change /var/model/taide-4bit.gguf to /var/model/<gguf file name>

  8. Edit run.sh:

Add <yaml file name> into confs array and execute run.sh again.
